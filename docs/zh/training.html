基于 GPTSoVITS 基准模型，微调训练与推理

使用场景 ​

如果3-6秒的声音样本无法满足你的克隆需求，可以尝试训练模型, 提升声音相似度和真实感；

应用预览 ​

微调训练 ​

请认真准备数据集，好的数据集是炼出好的模型的基础。

1. 训练数据准备 ​

格式 ​

json 文件内容格式

webapp/temp/test2/test2_zh.json
json
[
    ...
    {
        "idx": 4,
        "spk": 0,
        "lang": "en",
        "start": 9619,
        "end": 12145,
        "duration": 2526,
        "text": "曾经有一份真诚的爱情放在我面前，",
        "text_trans": "I was offered true love,",
        "speed": 1.0
    }
    ...
]

输入 ​

webapp/temp/test2/test2_xx.json

webapp/temp/test2/stems/test2_instrumental.wav

注意

  • 进入首页，上传音频或视频，
  • 打开 字幕 调试
  • 选择 提供商 及相应参数
  • 按需开启 音轨分离、声纹对齐、情感识别、说话人数量
  • 点击底部 执行

执行完毕后，在输出产物中可以得到 test2_xx.json 及 test_instrumental.wav

关于 test2_xx.json 文件

  • 不使用 音轨分离，可以单独上传处理后的音频，见下.
  • 不使用 声纹对齐，可以通过 Subtitle Edit 第三方软件进行手动 srt 文件对齐，并在提供商选择 srt 并上传
  • 不使用 情感识别，默认为 中立
  • 不使用 说话人数量，默认为 0，即为同1人
  • 建议
    • 超长音视频，选项 SRT、CapCut，可大大降低识别时间
    • 中文音视频，选项 FunAsr
    • 英文音视频，选项 FasterWhisper

关于 tes2_instrumental.wav 文件

  • 如果 test2_instrumental.wav 人声分离音质效果不好，可开启 uvr 应用中选择更多模型进行音轨分离
  • 默认使用 MDXC 架构 model_bs_roformer_ep_368_sdr_12.9628.ckpt 模型，vocals 的 SDR 值为 12.1
  • 查看 控制台 ，选择 stems 包含 vocals 轨道，其中SDR值越高越好，执行分离后并下载保存，可作为训练数据准备音频输入

如何控制情感?

目前2种方法：配置不同，效果不同，按需取舍；以 高兴 为例:

  • 通用型： 某角色训练数据包含但不限于 高兴，即默认
  • 定制型： 某角色训练数据只包含 高兴
  • 无指定： 则使用默认
webapp/data/sovits.json
json
{
    "name": "xxx",
    "gender": "Male",
    "locale": "zh-CN",
    "model": {
        "默认": {
            "gpt": "xxx-e15.ckpt",
            "vits": "xxx_e4_s32.pth"
        },
        "高兴": {
            "gpt": "xxx_happy-e15.ckpt",

    },
    "ref": {
        "默认": 10000000,
